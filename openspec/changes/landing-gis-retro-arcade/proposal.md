# Propuesta de cambio: Landing personal GIS retro arcade

## Contexto
Se requiere una landing personal en stack simple (HTML, CSS, JS) para marca GIS/geoespacial, con estética retro gaming arcade (8-bit/16-bit), predominio violeta y look profesional de portfolio.

## Problema
Sin una página de presentación consistente:
- no hay narrativa clara de servicios y proyectos geoespaciales,
- no existe una identidad visual diferenciada,
- se pierde capacidad de conversión para contacto.

## Objetivo
Construir una single-page landing en español que combine:
- claridad profesional de portfolio GIS,
- identidad retro arcade fuerte,
- buena experiencia responsive.

## Alcance
Incluye:
- Hero, Servicios, About, Proyectos, Stack, CTA final y Footer.
- Estilo retro arcade con UI pixelada, scanlines CRT, glow y HUD.
- Interacción mínima con JS (score HUD y navegación suave).

No incluye:
- Backend o formulario funcional con persistencia.
- CMS o panel admin.

## Enfoque
1. Estructura semántica HTML por secciones.
2. Sistema visual CSS con tokens de color y componentes arcade reutilizables.
3. JS liviano solo para microinteracción y navegación.

## Riesgos
- Exceso visual puede afectar legibilidad.
- Tipografías pixel pueden fatigar en párrafos largos.

## Mitigaciones
- Mantener tipografía de lectura separada de títulos.
- Asegurar contraste alto y layout limpio.

## Criterio de éxito
- La landing refleja claramente marca GIS + estética retro arcade.
- Se ve correctamente en mobile y desktop.
- CTA principal y navegación interna funcionan.
